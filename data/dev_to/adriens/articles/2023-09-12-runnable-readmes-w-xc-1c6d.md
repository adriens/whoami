## ❔ About

Sometimes, to describe what your project can do ( how to build/use it) you put script in your `README`... then the end user (including you) has to deal with a set of commands, put them in the right order,etc...

The aim of this post is to **show how to ease all this stuff**, right into the `README`, with [`xc`](https://github.com/joerdav/xc):

{% embed https://github.com/joerdav/xc %}

## 🪝 Teaser

In the demo, you'll se **how easy it is to implement and use tasks during a live coding session**.

## 🎯 Goal

The goal can be summarized as a single issue:

{% github https://github.com/adriens/auptitcafe-data/issues/4 %}

## 🧰 Stack

- [`xc`](https://xcfile.dev/) : to package `README` as **runnable tasks**
- [`tea.xyz`](https://tea.xyz/) : to easily install `duckdb` package
- [`GH Codespaces`](https://github.com/features/codespaces) : so we can **code the feature right into the cloud in our browser**... for the cloud
- [`duckdb`](https://duckdb.org/) : serverless & lighwight database
